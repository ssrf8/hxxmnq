# 模型状态投影契约

每轮只向正文模型投影当前场景必要信息，不发送完整 `stat_data`、完整角色库或数据库。

## 始终投影

- 当前日期、时段、季节、普通天气和异常天气；
- 玩家姓名、称谓、当前位置和庭园名称；
- 物资与灵感；
- 当前主要锚点；
- 当前交互会话或主要事件的短摘要。
- 当前交互目标的稳定 ID、入口行动 ID，以及 GAL `scene.v1` 所需的在场说话人白名单。

## 条件投影

- 只投影当前区域、关联设施及其状态；
- 只投影在场角色和本场相关角色；
- 每个相关角色只投影与当前参与者、设施或事件有关的有效关系事实；
- 只投影当前事件的前置、阻断、期限与允许结果；
- 战斗期间只投影选中的 `config_id`、叙事上下文和待结算结果；
- 数据库最多补充 3 条相关摘要，总量不超过约 1200 中文字符，并明确标记为只读回忆。

## 不向模型投影

- UI 草稿、动画帧、画布坐标、心跳和诊断日志；
- GAL 当前片段索引、立绘文件路径和本地素材注册表；
- 完整数据库、无关角色档案、全部历史事件；
- 已归档的失效关系事实，除非当前剧情明确回顾；
- 高权限 API、脚本源码、远程资源地址。

## 防全知

投影给模型不等于角色知道。正文必须按目击、告知、调查、推断或既有经历判断每个角色的知识来源；数据库摘要和系统状态不能直接变成角色记忆。
