# 动态开场事务契约

0 号楼设置页只维护本地草稿：姓名、称谓/代词、外貌简述、庭园名称。预览和修改草稿不得写入 MVU。

确认时：

1. 校验并冻结草稿快照；
2. 生成一条可编辑的真实 user 开场消息；
3. 通过正常发送与生成链提交，不直接替换 `stat_data`；
4. 正文模型描述玩家抵达荒废庭园并开始自由观察；
5. 变量模型在同一正常回复中写入玩家资料、庭园名，将 `meta.initialized` 与 `meta.opening_committed` 置为 true；
6. 若生成失败，保持 opening 未提交并允许安全重试；禁止产生重复 user 消息。

快速开始优先读取 Persona。缺失字段使用“中性称谓”和“无名庭园”，外貌可为空。叙事人称由玩家预设和实际输入决定。

验收必须覆盖：新聊天、刷新后重开、失败重试、重复点击、Swipe 分支，以及实际消息楼层中的持久状态。
