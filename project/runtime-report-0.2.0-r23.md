## 本轮根因与修复

R20 温室链的正式状态由模型自由输出更新，导致“事件完成、区域未解锁、设施仍可建设”的矛盾状态。R23 将受控选项改为两次带预设请求：第一次沿用酒馆当前预设生成可见剧情；第二次使用 `generate({ preset_name: 'in_use', should_silence: true, json_schema })` 只返回登记结果。结果必须通过事件与动作白名单，再由本地结算器从请求前状态计算并一次写入同一真实 assistant 楼层，随后复读验证。失败或空回复保持可重试事务，不重复创建玩家楼层。

真实运行检查又发现游戏 iframe 宿主没有转发新增的 `generate`。R23 在 `ui-host-shell.js` 中补齐该 API，并增加契约测试，避免双请求代码在 iframe 内不可达。

## 继续验收

1. 选择“幻想乡物语·移动庭园（测试检查点 0.2.0-r23）”，新建聊天；确认出现“在穿过结界之前”开场表单。
2. 填写开场并进入庭院；确认此步不调用模型，灵梦在庭院可见。
3. 点击灵梦，选择“检查结界”。该动作会依次发生剧情生成与静默结算解析，等待时间会比旧版长；完成后应能继续庭园行动，不能出现空白卡死。
4. 继续温室链：调查魔力痕迹 → 获取第二点灵感 → 清理地基 → 建造温室。魔理沙线索结算后，温室区域解锁、设施状态与关键事件必须同时一致。
5. 若第二次请求或写入失败，界面应显示“重试本地结算”；点击重试不得新增重复玩家楼层。

不要继续使用 R20、R21 或 R22 中间包。
