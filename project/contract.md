# 稳定契约

## 必须成立

- `stat_data` 是唯一正式游戏状态源。
- 数据库只做可选长期归档与检索。
- 玩家只在符卡战中显示，不出现在庭园地图。
- 第一版固定角色为灵梦、魔理沙、琪露诺、爱丽丝、米斯蒂娅、萃香、荷取、咲夜。
- 角色交互允许跨越多轮消息，并以显式结束完成结算。
- 玩家正常游玩时只看见一个常驻游戏壳；该壳不绑定真实消息 0 楼，后台真实消息只做视觉隐藏。
- GAL 剧情必须镜像真实 SillyTavern user/assistant 楼层；自定义输入、建议回应和结束聊天均走正常消息事务。
- 角色和设施的入口操作由本地受控配置与当前 MVU 状态计算，LLM 不得临时创造可执行按钮或绕过前置条件。
- 一次 assistant 回复最多渲染 6 个 GAL 演出片段；片段全部播放后才允许玩家提交下一次回应。
- 结束聊天必须先完成一次真实收尾生成和幂等 MVU 结算；完成一次交互回合不等于必然推进时段。
- 角色立绘由本地素材注册表解析；LLM 只可选择白名单角色、反应和姿态标签。
- 变量更新只使用目标 MVU 支持的 JSONPatch `add`、`replace`、`remove`。
- 妖花战结果只允许本地 bridge 在白名单、范围和事件前置校验后写入 `battle.current`；模型只能消费该字段，正文中的第二份结果无效。
- 战斗结算以 `settlement_id` 幂等；四种允许结果都只留下移动锚点线索，r20 不创建正式稳定锚点。
- 新开场只允许 bridge 在首个 assistant 楼层执行一次确定性 MVU 写入；必须复读成功后进入庭院，不得调用 LLM 初始化资料。
- UI 故障时必须恢复原生聊天。
- 游戏壳心跳丢失、脚本卸载、切换角色卡或进入维护模式时，必须恢复被视觉隐藏的原生楼层和输入区。
- 角色像素素材默认闭嘴、中性或轻微微笑。

## 禁止成立

- 大妖精不得作为第一版固定角色。
- 不得把玩家作为庭园地图行走角色。
- 不得使用 `is_hidden` 改变真实消息上下文。
- 不得把 GAL 片段、当前立绘、点击进度或输入草稿写成第二份正式剧情状态。
- 不得将模型输出作为 HTML 插入页面，不得执行模型给出的图片路径、URL、脚本或未登记操作。
- UI 和数据库不得绕过 MVU 成为第二事实源。
- 不得让新开场依赖 LLM 输出玩家资料、庭园名或初始化变量块。
- 不得执行 LLM 生成的代码、URL、HTML 或未校验战斗配置。
- 不得从模型正文、玩家声称的 JSON 或任意 HTML 覆盖 `battle.current`，不得重复消费已在 `battle.settled_ids` 中的结果。
- 不得在未授权时打包、发布或覆盖角色卡成品。
